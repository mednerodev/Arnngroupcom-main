═══════════════════════════════════════════════════════════════
  STEP 1: GO TO VERCEL DASHBOARD
═══════════════════════════════════════════════════════════════

1. Open: https://vercel.com/dashboard
2. Find and click your project: "Arnngroupcom"
3. Click "Settings" (top navigation)
4. Click "General" (left sidebar)
5. Scroll to "Build & Development Settings"

═══════════════════════════════════════════════════════════════
  STEP 4: REDEPLOY
═══════════════════════════════════════════════════════════════

Option A - Auto Redeploy (Recommended):
   1. Push your code to GitHub
   2. Vercel will auto-deploy with new settings
   
   Commands:
   git add .
   git commit -m "Update Vercel configuration"
   git push origin main

Option B - Manual Redeploy:
   1. Go to "Deployments" tab
   2. Click the latest deployment
   3. Click "..." (three dots menu)
   4. Click "Redeploy"
   5. Click "Redeploy" to confirm

═══════════════════════════════════════════════════════════════
  TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

If it STILL fails:

1. Check build logs for errors
2. Ensure Output Directory says "build" not "dist"
3. Try clearing build cache:
   - Deployments → ... → Redeploy
   - Uncheck "Use existing Build Cache"
   - Click Redeploy

4. Verify vite.config.ts has: outDir: 'build'
